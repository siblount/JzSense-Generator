Detailed Description

A DAZScript wrapper for QDomNode. This is an abstract base class that provides general functionality for Document Object Model (DOM) nodes.

For more information about the Document Object Model (DOM) see the Level 1 and Level 2 Core specifications.

For a more general introduction of the DOM implementation see the DzDomDocument documentation.

Properties

String : localName

Holds the local name of this node, if the node uses namespaces; only DzDomElement or DzDomAttr nodes can have namespaces; a namespace must have be specified at creation time - it is not possible to add a namespace post-creation.. (Read Only)

String : namespaceURI

Holds the namespace URI of this node. (Read Only)

String : nodeName

Holds the name of this node - the meaning of which depends on nodeType : (Read Only) * for AttributeNode - the name of the attribute

  • for CDATASectionNode - the string “#cdata-section”
  • for CommentNode - the string “#comment”
  • for DocumentNode - the string “#document”
  • for DocumentFragmentNode - the string “#document-fragment”
  • for DocumentTypeNode - the name of the document type
  • for ElementNode - the tag name
  • for EntityNode - the name of the entity
  • for EntityReferenceNode - the name of the referenced entity
  • for NotationNode - the name of the notation
  • for ProcessingInstructionNode - the target of the processing instruction
  • for TextNode - the string “#text”

NodeType : nodeType

Holds the type of this node. (Read Only)

String : nodeValue

Holds the value of this node - the meaning of which depends on nodeType : * for AttributeNode - the attribute value

  • for CDATASectionNode - the contents of the CDATA section
  • for CommentNode - the comment
  • for ProcessingInstructionNode - the data of the processing instruction
  • for TextNode - the text
  • for all others - an empty string

String : prefix

Holds the namespace prefix of this node; only DzDomElement or DzDomAttr nodes can have namespaces; a namespace prefix must have be specified at creation time - it is not possible to add a namespace post-creation.
